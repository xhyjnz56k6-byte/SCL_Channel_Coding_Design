# 三类信道编码公共基础仿真规划（码率定义修订版）

## 1. 规划目的

本规划用于统一 BCH、卷积码和 LDPC 三类编码的公共仿真基础，冻结以下内容：

- 随机信息帧生成；
- 每帧独立随机噪声；
- BPSK、AWGN 和 LLR 定义；
- 码率、BER、FER、编码增益和时延定义；
- smoke、prescan、formal 停止规则；
- 曲线图片输出及命名；
- 交织启用条件；
- 三类编码的接口和目录边界；
- 卷积码硬/软判决 Viterbi 对比；
- LDPC 码长、码率灵活调整以及 BP/NMS 对比。

公共基础阶段不实现三类码的具体编译码算法，只建立可复用、可验证、可复现的仿真框架。

---

## 2. 老师最新要求冻结

### 2.1 噪声

- 每一帧都必须重新随机生成噪声；
- 不允许不同帧反复使用同一条噪声向量；
- 同一帧内不同码元的噪声样本相互独立；
- 相同 seed、case、SNR 和 frame index 下必须能够复现；
- 同一算法对比必须使用相同的接收数据。

### 2.2 BCH

- BCH(15,11,1) 分块方案采用 syndrome 查表译码；
- 查表译码只用于分块 BCH；
- 整块缩短 BCH 采用 BM + Chien 搜索译码；
- 整块 BCH 不使用查表译码。

### 2.3 卷积码

必须同时实现：

- 硬判决 Viterbi；
- 软判决 Viterbi。

两种译码器必须使用相同的原始输入帧、编码结果、接收符号、噪声、SNR、码率和停止规则。

### 2.4 交织

- 交织只在突发错误场景下测试；
- 基础 AWGN 曲线不加入交织；
- BCH 和卷积码进行交织对比；
- LDPC 不配置交织；
- 交织结果单独汇总，不与基础编码性能混在同一结论中。

### 2.5 曲线

- 所有仿真曲线必须保存为图片；
- 图片名称必须包含阶段、码型、case、信道、译码算法和指标；
- 禁止使用 `figure1.png`、`result.png` 等无意义名称；
- smoke、prescan、formal 的图片和结果必须分目录保存；
- 图片不得覆盖旧 Stage 的结果。

### 2.6 LDPC

- 固定采用 BG2；
- 支持目标码长 480 bit、576 bit 和不超过 640 bit 的扩展码长；
- 需要考虑码长和码率的灵活调整机制；
- 对比 Layered BP 和 Layered NMS；
- 记录目标码长、实际码长、实际码率、提升因子、填充和打孔信息。

## 3. 统一码率定义

三类编码的码率统一定义为：

\[
R=\frac{\text{原始输入长度}}{\text{编码后长度}}
\]

记为：

\[
R=\frac{K_{\mathrm{input}}}{N_{\mathrm{encoded}}}
\]

其中：

- \(K_{\mathrm{input}}\)：编码前的原始信息长度；
- \(N_{\mathrm{encoded}}\)：该方案最终编码输出的实际长度；
- 分块编码时，分母为所有编码块拼接后的总长度；
- 卷积码加入尾比特后，分母包含尾比特编码产生的输出；
- 卷积码打孔后，分母使用打孔后的实际编码输出长度；
- LDPC 填充位不增加分子，分母使用最终实际编码输出长度；
- 不使用母码理论码率代替实际仿真码率。

示例：

- BCH 分块 200 bit：\(R=200/285\approx0.70175\)；
- BCH 整块 200 bit：\(R=200/248\approx0.80645\)；
- 卷积码 300 bit、1/2 零尾：\(R=300/612\approx0.49020\)；
- LDPC 300 bit、实际输出 480 bit：\(R=300/480=0.625\)。

若 LDPC 实际码长与目标码长不同，必须按实际编码后长度重新计算码率。

## 6. 公共随机信息帧

### 6.1 帧长度

建立两类公共输入帧：

```text
K = 200 bit
K = 300 bit
```

| 编码 | 200 bit | 300 bit |
|---|---:|---:|
| BCH | 正式使用 | 正式使用 |
| 卷积码 | 程序兼容 | 正式主场景 |
| LDPC | 可作为扩展 | 正式主场景 |

### 6.2 随机生成

每一帧信息位独立生成：

\[
u_{f,i}\in\{0,1\},\qquad P(0)=P(1)=0.5
\]

同一个 `frameIndex` 在三类编码中对应同一条原始输入信息。

### 6.3 扩展到 50000 帧

公共帧池按 shard 保存，例如：

```text
frames_000000_009999.bin
frames_010000_019999.bin
frames_020000_029999.bin
frames_030000_039999.bin
frames_040000_049999.bin
```

规则：

- 旧 shard 永不修改；
- 新增帧只追加新 shard；
- smoke 读取前 100 帧；
- prescan 最多读取前 2000 帧；
- formal 按停止规则最多读取 50000 帧；
- manifest 记录 seed、总帧数、格式版本和 SHA256；
- 扩展后必须验证旧 shard 哈希未变化。

## 7. 每帧独立随机噪声机制

### 7.1 数学模型

对于 SNR 点 \(s\)、帧 \(f\)、码元 \(i\)：

\[
z_{s,f,i}\sim\mathcal N(0,1)
\]

接收符号：

\[
y_{s,f,i}=x_{f,i}+\sigma_s z_{s,f,i}
\]

必须满足：

- 不同帧的噪声向量不同；
- 同一帧不同码元的噪声独立；
- 不同 SNR 点使用不同噪声流；
- 相同参数下可以精确复现。

### 7.2 seed 分层

建议分为：

```text
masterSeed
payloadSeed
noiseSeed
interleaverSeed
```

噪声 seed 由以下内容共同确定：

```text
masterSeed
experimentId
caseId
snrIndex
frameIndex
```

推荐使用：

```cpp
std::seed_seq seedSeq{
    masterSeed,
    experimentId,
    caseId,
    snrIndex,
    frameIndex
};

std::mt19937 generator(seedSeq);
std::normal_distribution<double> gaussian(0.0, 1.0);
```

每一帧都重新构造该帧对应的生成器并生成新的标准高斯噪声。

### 7.3 公共母噪声长度

建议每帧生成：

```text
motherNoiseLength = 1000
```

不同方案截取前 \(N_{\mathrm{encoded}}\) 个样本。

示例：

```text
BCH(248,200)       → 前 248 个
BCH 分块 200       → 前 285 个
BCH(390,300)       → 前 390 个
BCH 分块 300       → 前 420 个
卷积码             → 按实际编码后长度截取
LDPC               → 前 480、576 或实际不超过 640 个
```

### 7.4 公平比较

卷积码：

```text
同一 receivedSymbols
├─ 硬判决 → 硬判决 Viterbi
└─ 软信息 → 软判决 Viterbi
```

LDPC：

```text
同一 LLR
├─ Layered BP
└─ Layered NMS
```

### 7.5 噪声测试

必须完成：

1. 同帧复现测试；
2. 不同帧噪声不同测试；
3. 均值接近 0；
4. 方差接近 1；
5. checkpoint/resume 结果一致；
6. 固定母噪声文件可被 MATLAB 和 C++ 一致读取；
7. 保存 frame 0、1、2 的前若干噪声样本供人工检查。

Gate：

```text
PASS_RANDOM_NOISE_PER_FRAME
```

## 8. BPSK、AWGN 和 LLR

### 8.1 BPSK

\[
0\rightarrow+1,\qquad 1\rightarrow-1
\]

### 8.2 AWGN 噪声方差

统一使用：

\[
R=\frac{K_{\mathrm{input}}}{N_{\mathrm{encoded}}}
\]

当 BPSK 符号能量为 1 时：

\[
\sigma^2=\frac{1}{2R10^{E_b/N_0/10}}
\]

### 8.3 LLR

\[
L_i=\ln\frac{P(c_i=0|y_i)}{P(c_i=1|y_i)}=\frac{2y_i}{\sigma^2}
\]

约定：

```text
LLR > 0：更倾向于 bit 0
LLR < 0：更倾向于 bit 1
```

## 10. 统一统计指标

### 10.1 BER

只统计原始输入信息位：

\[
BER=\frac{\text{原始信息错误比特数}}{\text{原始信息总比特数}}
\]

不统计 filler、尾比特和校验位。

### 10.2 FER

\[
FER=\frac{\text{错误帧数}}{\text{总帧数}}
\]

只要一帧原始信息中至少有 1 bit 错误，该帧即为错误帧。

### 10.3 译码成功率

\[
P_{\mathrm{success}}=1-FER
\]

### 10.4 编码增益

编码增益是正式输出指标，优先在 `FER = 10^-2` 或双方曲线均覆盖的统一目标点计算：

\[
G_{\mathrm{coding}}=(E_b/N_0)_{\mathrm{reference}}-(E_b/N_0)_{\mathrm{candidate}}
\]

若未覆盖目标点，则输出：

```text
codingGain = N/A
```

### 10.5 时延

分别统计：

- 平均编码时间；
- 最大编码时间；
- 平均译码时间；
- 最大译码时间；
- 可选 P95 译码时间。

算法时延不包含文件 IO、图片绘制、日志打印、公共帧读取和随机噪声生成。

### 10.6 复杂度

至少记录：

- 理论复杂度；
- 每帧主要操作量；
- 内存使用；
- BCH 查表规模；
- Viterbi 状态数和 ACS 次数；
- LDPC 边数和迭代次数；
- BP 与 NMS 的复杂度差异。

## 13. LDPC 码长和码率灵活调整

### 13.1 固定条件

```text
BG = 2
Kinput = 300 bit 为正式主场景
targetN = 480 / 576 / ≤640 bit
decoder = Layered BP / Layered NMS
```

### 13.2 case 参数

每个 LDPC case 至少记录：

```text
BG
Zc
kb
nb
mb
originalInputLength
Keff
filler
shortenCount
punctureCount
encodedLength
codeRate
decoder
maxIterations
alpha
```

其中：

\[
\text{codeRate}=\frac{\text{originalInputLength}}{\text{encodedLength}}
\]

### 13.3 灵活码长选择流程

```text
给定原始输入长度
→ 给定目标码长 480 / 576 / ≤640
→ 枚举合法 Zc
→ 枚举允许保留的 BG2 列数
→ 计算系统容量 Keff
→ 检查 Keff 是否容纳原始输入
→ 计算 filler 和候选编码后长度
→ 检查实际长度是否接近目标
→ 构造 H = [Hu Hp]
→ 检查 Hp 满秩
→ 选择最优候选
```

选择优先级：

1. 实际编码后长度等于目标码长；
2. 不能精确达到时，选择距离目标最近的合法码长；
3. 实际编码后长度不超过 640 bit；
4. \(H_p\) 满秩；
5. filler 尽量少；
6. Tanner 图结构正常；
7. 明确记录目标值和实际值偏差。

### 13.4 码率调整优先级

```text
调整保留列数
→ 调整 Zc
→ 填充/缩短
→ 必要时打孔
```

### 13.5 BP 与 NMS

同一 LDPC case 必须共享同一 H、同一输入、同一码字、同一噪声、同一 LLR 和同一最大迭代次数。

NMS 归一化系数 prescan 建议：

```text
alpha ∈ {0.70, 0.75, 0.80, 0.85, 0.90}
```
