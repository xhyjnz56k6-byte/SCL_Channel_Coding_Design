# Repository-verified Common integration contract

All names below are existing `scl::common` APIs. Read a `PayloadFrame` through `PackedFramePoolReader` (`common/frame_pool.hpp`), retain `frameIndex`, and give `payloadBits` to the future BCH adapter. The adapter yields `BitVector` coded bits and frozen `CodeLengths`. Modulate with `bpskModulate` (`common/modulation.hpp`). Read an identically indexed standard-noise prefix from `NoisePoolReader::readFramePrefix` (`common/noise_pool.hpp`), calculate sigma with `computeAwgnSigma`, and form received symbols through `applyAwgn` (`common/awgn_channel.hpp`). Call `hardDecision` (`common/demodulation.hpp`) and pass `HardBitInput` to the future lookup decoder. It returns `DecodeResult` (`common/result_types.hpp`); trim filler before `addFrameResult` and `summarizeMetrics` (`common/simulation_metrics.hpp`).

The common summary target is `SummaryRow` (`common/result_schema.hpp`): use `codeType=BCH`, case name, frozen payload/encoded lengths, rate, SNR, `ErrorMetrics`, stop reason, frame/noise pool IDs, and config hash. BCH-only audit fields belong in `bch_detail.csv`: component parameters, organization, block count, filler bits, decoder type/status, syndrome, corrected position, and recovery/trimming status. For a zero syndrome, record `NO_ERROR` and leave corrected position empty; the 15-entry lookup table is only for nonzero syndromes. The current public `SummaryRow` has no case-id field; include the case identity in `caseName` and the canonical BCH configuration text used to make `configHash`.

Checkpoint uses `SimulationCheckpointRecord` (`common/checkpoint.hpp`) with payload/encoded lengths, pool IDs, SNR, metrics, and `configHash`. The future BCH runner must include case identity, filler, organization, decoder type, and convention version in its canonical configuration before calling `computeConfigHash`, so resume rejects a different BCH case.
