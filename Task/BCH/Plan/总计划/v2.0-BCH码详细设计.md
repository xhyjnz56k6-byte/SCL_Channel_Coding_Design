# SCL_Channel_Coding_Design：BCH 码详细设计与仿真实施规划

> 本文件综合既有 BCH 设计方案、老师提供的《信道编码、交织与译码方案及仿真分析》、项目目录与 Git 工作流约束，以及本轮最新确认内容形成。  
> 本文件作为后续 BCH 编码、译码、AWGN 仿真、复杂信道与交织实验的统一执行依据。

# 1. 项目背景与 BCH 阶段定位

项目名称：

```text
SCL_Channel_Coding_Design
```

整体研究顺序：

```text
BCH
→ 卷积码 CC
→ LDPC
```

当前阶段只推进 BCH，不进入卷积码与 LDPC。

老师对低速电文的总体要求为：

- 低速电文只采用 BCH；
- 输入信息长度覆盖 200 bit 和 300 bit；
- 比较 BCH(15,11,1) 分块方案与整块缩短 BCH；
- 单个编码块编码后总码长不超过 1000 bit；
- 输出 BER、FER、译码成功率、译码时延、编码增益、复杂度；
- 交织作为独立测试项，不是 BCH 固定链路；
- 后续扩展 AWGN、突发错误、短时遮挡、多径、频偏等信道；
- 分块 BCH 使用 syndrome 查表译码；
- 整块 BCH 使用 BM + Chien 代数译码；
- AWGN 中每一帧随机生成新的高斯噪声，不允许所有帧复用同一个噪声向量。

---

# 2. 本次 BCH 阶段的最终目标

BCH 阶段需要完成两条独立链路：

```text
主线 A：BCH(15,11,1) 分块方案
主线 B：整块缩短 BCH 方案
```

最终回答以下问题：

1. 200 bit 和 300 bit 低速电文采用 BCH(15,11,1) 分块后，需要多少块、多少填充、编码后码长是多少；
2. 分块 BCH 的单块纠错能力如何影响完整电文 FER；
3. 分块 BCH 与整块缩短 BCH 在 BER、FER 和译码成功率方面的差异；
4. 分块查表译码与整块 BM + Chien 译码在复杂度和时延方面的差异；
5. 两类方案在 AWGN 下的 waterfall 区域与正式曲线；
6. 两类方案相对未编码 BPSK 的编码增益；
7. 交织对突发错误条件下 BCH 性能的改善；
8. 最终形成面向 200 bit 和 300 bit 低速电文的工程选型结论。

# 4. 码率定义与噪声计算规则

## 4.1 BCH 码率定义

本项目 BCH 统一定义：

\[
R=rac{K}{N}
\]

其中：

- \(K\)：输入信息长度；
- \(N\)：编码后码长。

统一用词：

```text
输入信息长度
编码后码长
编码码率
```

不使用“发送长度决定码率”的措辞。

## 4.2 四个 Case 的码率

| Case | 输入信息长度 K | 编码后码长 N | 编码码率 R |
|---|---:|---:|---:|
| BCH-S200 | 200 | 285 | 0.701754 |
| BCH-S300 | 300 | 420 | 0.714286 |
| BCH-B200 | 200 | 248 | 0.806452 |
| BCH-B300 | 300 | 390 | 0.769231 |

## 4.3 AWGN 噪声方差

BPSK 映射：

\[
0ightarrow +1,\qquad 1ightarrow -1
\]

AWGN：

\[
y_i=x_i+\sigma z_i
\]

其中：

\[
z_i\sim\mathcal N(0,1)
\]

\[
\sigma^2=
rac{1}
{2R10^{E_b/N_0/10}}
\]

每个 Case 使用自己的 \(R=K/N\)。

# 5. 每帧随机噪声策略

老师要求：

```text
噪声必须随机；
每一帧重新生成新的随机噪声。
```

## 5.1 正确实现方式

对于第 \(f\) 帧、第 \(i\) 个编码符号：

\[
z_{f,i}\sim\mathcal N(0,1)
\]

\[
y_{f,i}=x_{f,i}+\sigma z_{f,i}
\]

必须满足：

- 不同帧的噪声向量不同；
- 同一帧内不同符号的噪声样本不同；
- 相同随机种子重跑时结果可复现；
- 不同随机种子产生不同实验结果。

## 5.2 推荐随机数实现

每个 Case 和每个 SNR 点初始化一次随机数生成器：

```cpp
std::mt19937_64 rng(seed);
std::normal_distribution<double> gaussian(0.0, 1.0);
```

随后在帧循环中持续生成：

```cpp
for each frame:
    for each encoded symbol:
        z = gaussian(rng)
        y = x + sigma * z
```

禁止：

```text
每一帧都用同一个 seed 重新初始化 RNG
所有帧复用同一固定噪声向量
正式 BER/FER 实验读取一个固定噪声帧反复使用
```

## 5.3 公共噪声目录的定位

`Task/Common` 中共享的是：

```text
随机数生成器类型
master seed
seed 派生规则
高斯分布定义
Eb/N0 到 sigma 的计算公式
随机调用顺序
复现规范
```

不共享一个固定噪声向量。

固定噪声只允许用于：

```text
MATLAB/C++ 数值逐元素对比
bug 复现
固定 trace
单元测试
```

不用于正式 BER/FER 性能曲线。

# 6. BCH(15,11,1) 分块方案设计

## 6.1 固定参数

| 参数 | 配置 |
|---|---|
| 码型 | BCH(15,11,1) |
| 信息位 | 11 bit/组 |
| 码长 | 15 bit/组 |
| 纠错能力 | \(t=1\) |
| 生成多项式 | \(g(x)=x^4+x+1\) |
| 编码结构 | 系统码 |
| 译码算法 | syndrome 查表 |
| 不足一组 | 尾部补 0 |
| 译码后 | 去除 filler |

## 6.2 系统编码

\[
c(x)=x^{n-k}m(x)+r(x)
\]

其中：

\[
r(x)=x^{n-k}m(x)mod g(x)
\]

对于 BCH(15,11,1)：

\[
n-k=4
\]

推荐冻结位序：

```text
message[0]  ↔ x^10
message[10] ↔ x^0

codeword[0]  ↔ x^14
codeword[14] ↔ x^0

系统码排列：
[11 bit 信息位][4 bit 校验位]
```

该位序必须由固定向量和 MATLAB 参考共同确认后冻结。

## 6.3 200 bit 分块

\[
\left\lceilrac{200}{11}ightceil=19
\]

\[
F=19	imes11-200=9
\]

\[
N=19	imes15=285
\]

流程：

```text
200 bit payload
→ 尾部补 9 个 0
→ 形成 209 bit
→ 分成 19 个 11 bit 信息块
→ 每块编码成 15 bit
→ 拼接得到 285 bit 编码帧
```

## 6.4 300 bit 分块

\[
\left\lceilrac{300}{11}ightceil=28
\]

\[
F=28	imes11-300=8
\]

\[
N=28	imes15=420
\]

流程：

```text
300 bit payload
→ 尾部补 8 个 0
→ 形成 308 bit
→ 分成 28 个 11 bit 信息块
→ 每块编码成 15 bit
→ 拼接得到 420 bit 编码帧
```

# 7. 分块 BCH 查表译码设计

## 7.1 查表译码只用于分块 BCH

查表译码仅适用于：

```text
BCH(15,11,1)
```

整块 BCH 不使用查表译码。

## 7.2 syndrome 表

对于 15 个单比特错误位置，建立：

```text
15 个非零 syndrome
↔
15 个错误位置
```

必须验证：

- syndrome 0 只表示无错；
- 15 个单错 syndrome 均非零；
- 15 个单错 syndrome 互不相同；
- 每个 syndrome 唯一映射到一个错误位置；
- 错误位置编号与码字位序一致。

## 7.3 译码流程

```text
接收 15 bit 码字
        ↓
计算 syndrome
        ↓
syndrome = 0？
   ├─ 是：NO_ERROR
   └─ 否：查 syndrome 表
               ↓
          定位单比特错误
               ↓
          翻转对应 bit
               ↓
          再次计算 syndrome
               ↓
          提取 11 bit 信息位
```

## 7.4 译码状态

推荐：

```text
NO_ERROR
CORRECTED_SINGLE_ERROR
POST_CHECK_FAILED
UNRECOGNIZED_SYNDROME
```

注意：

- 对于 BCH(15,11,1)，双错可能被误纠成另一个合法码字；
- 因此不能简单把所有双错都标为 `UNCORRECTABLE`；
- 必须通过原始码字对比进行误纠审计。

# 8. 整块缩短 BCH 设计

## 8.1 200 bit 整块方案

计划：

```text
母码：BCH(255,207)
缩短后：(248,200)
```

缩短位数：

\[
s=207-200=7
\]

缩短后：

\[
n=255-7=248
\]

## 8.2 300 bit 整块方案

计划：

```text
母码：BCH(511,421)
缩短后：(390,300)
```

缩短位数：

\[
s=421-300=121
\]

缩短后：

\[
n=511-121=390
\]

## 8.3 缩短编码流程

```text
原始 payload
→ 在冻结位置补已知 0
→ 形成母码信息向量
→ 按母 BCH 系统编码
→ 删除补入的已知系统位
→ 得到缩短 BCH 码字
```

## 8.4 缩短译码流程

```text
收到缩短码字
→ 在冻结位置补回已知 0
→ 计算 syndrome 序列
→ Berlekamp-Massey
→ 得到错误位置多项式
→ Chien 搜索
→ 定位错误位置
→ 翻转纠错
→ 再次校验
→ 删除补回位
→ 恢复 payload
```

## 8.5 整块参数必须先冻结

正式实现前必须核对：

```text
母码 n0
母码 k0
纠错能力 t
primitive polynomial
generator polynomial
缩短位置
缩短位数
MATLAB 对应参数
bit ordering
系统码排列
```

未经验证不得直接开始 BM + Chien 正式实现。

# 9. 统一指标定义

## 9.1 BER

只统计原始 payload：

\[
\mathrm{BER}
=
rac{	ext{错误 payload bit 总数}}
{	ext{发送 payload bit 总数}}
\]

不统计：

```text
filler
校验位
缩短补回位
母码虚拟位
```

## 9.2 FER

一帧 200 bit 或 300 bit payload 中，只要存在至少一个错误 bit：

```text
frameError = 1
```

\[
\mathrm{FER}
=
rac{N_{\mathrm{frameError}}}
{N_{\mathrm{frames}}}
\]

对于分块 BCH：

```text
19 或 28 个小块中，
只要任一小块最终恢复错误，
完整电文即计为误帧。
```

## 9.3 真实译码成功率

\[
P_{\mathrm{trueSuccess}}
=
rac{	ext{payload 完全正确帧数}}
{	ext{总帧数}}
\]

## 9.4 译码器报告成功率

\[
P_{\mathrm{reportedSuccess}}
=
rac{	ext{译码器未报告失败帧数}}
{	ext{总帧数}}
\]

两者差异用于发现：

```text
误纠
未检测错误
状态判定失真
```

## 9.5 误纠率

\[
P_{\mathrm{miscorrection}}
=
rac{	ext{错误输入被译成另一合法码字的次数}}
{	ext{超出纠错能力测试总数}}
\]

## 9.6 编码增益

选择：

```text
FER = 10^-2
```

定义：

\[
G_{\mathrm{coding}}
=
\left(rac{E_b}{N_0}ight)_{\mathrm{reference}}
-
\left(rac{E_b}{N_0}ight)_{\mathrm{candidate}}
\]

报告：

```text
分块 BCH 相对未编码 BPSK
整块 BCH 相对未编码 BPSK
整块 BCH 相对同 payload 分块 BCH
```

若曲线未覆盖目标 FER：

```text
N/A
```

禁止远距离外推。
