但不能仅根据：

$$


### BCH-B300

计划母码：

```text

```

```text
m = 9
n0 = 511
k0 = 421
parityLength = 90
shorteningLength = 121
shortenedN = 390
shortenedK = 300
```

同样必须通过 MATLAB 确认：

- 该母码参数是否属于合法 primitive narrow-sense BCH；
- MATLAB 是否直接支持该参数；
- 真实纠错能力是多少；
- generator degree 是否正好为 90；
- 是否需要使用自定义生成多项式；
- 是否与老师规划中的 `(390,300)` 一致。

## 4.3 缩短位置冻结

建议采用：

```text
母码系统排列：
[母码信息位][母码校验位]

缩短方式：
在母码信息位最前端补 s 个已知 0；
完成母码系统编码；
发送时删除这 s 个系统位。
```

例如 BCH-B200：

```text
200 bit payload
→ 前面补 7 个 0
→ 207 bit 母码信息
→ 编码为 255 bit
→ 删除前 7 个系统位
→ 248 bit 缩短码字
```

BCH-B300：

```text
300 bit payload
→ 前面补 121 个 0
→ 421 bit 母码信息
→ 编码为 511 bit
→ 删除前 121 个系统位
→ 390 bit 缩短码字
```

这里必须冻结“前端补零”还是“尾部补零”。后续 C++ 与 MATLAB 必须完全一致。

## 4.4 GF 模块设计

建议目录：

GF 至少支持：

```text
zero
one
addition
subtraction
multiplication
division
inverse
power
alphaPower
log
antilog
polynomial evaluation
```

在特征 2 的有限域中：

$$
a+b=a-b
$$

因此加法和减法均是 XOR。

乘法可以使用 log/antilog 表：

$$
a\cdot b
=
\alpha^{\log_\alpha(a)+\log_\alpha(b)}
$$

除法：

$$
\frac{a}{b}
=
\alpha^{\log_\alpha(a)-\log_\alpha(b)}
$$

## 4.5 GF 测试

$$

$$

$$

$$

$$

$$

$$

$$

$$
a\cdot a^{-1}=1,\qquad a\ne0

# 五、BCH-08：整块缩短系统编码器

建议分支：

```text
bch-08-block-shortened-encoder
```

最终 Gate：

```text
PASS_BCH08_BLOCK_SHORTENED_ENCODER
```

## 5.1 目标

实现：

```text
BCH-B200：
200 bit → 248 bit

BCH-B300：
300 bit → 390 bit
```

## 5.2 母码系统编码

设母码为：

$$
(n_0,k_0)
$$

生成多项式为：

$$
g(x)
$$

校验长度：

$$
r=n_0-k_0
$$

系统编码：

$$
c(x)=x^r m(x)+
\left[x^r m(x)\bmod g(x)\right]
$$

## 5.3 缩短编码流程

```text
payload
    ↓
在冻结位置补 s 个 0
    ↓
构造 k0 bit 母码信息
    ↓
母码系统编码
    ↓
检查 motherCodeword mod g(x)=0
    ↓
删除 s 个虚拟系统位
    ↓
得到 shortened codeword
```

## 5.4 数据结构

建议：

```cpp
struct BlockBchProfile {
    std::string caseName;
    std::size_t payloadLength;
    std::size_t motherN;
    std::size_t motherK;
    std::size_t shorteningLength;
    std::size_t shortenedN;
    std::size_t shortenedK;
    unsigned fieldDegree;
    unsigned correctionCapability;
    std::uint32_t primitivePolynomial;
    BitVector generatorPolynomial;
};
```

编码结果：

```cpp
struct BlockBchEncodeResult {
    BitVector payload;
    BitVector motherInformation;
    BitVector motherCodeword;
    BitVector shortenedCodeword;
    std::size_t shortenedPrefixLength;
};
```

## 5.5 测试矩阵

每个 Case 至少覆盖：

```text
全零
全一
首位单 1
末位单 1
0101 交替
1010 交替

```

# 六、BCH-09：BM + Chien 整块代数译码器

建议分支：

```text
bch-09-block-bm-chien-decoder
```

最终 Gate：

```text
PASS_BCH09_BLOCK_ALGEBRAIC_DECODER
```

## 6.1 译码主流程

```text
shortened received word
        ↓
补回 s 个已知 0
        ↓
恢复 n0 bit mother received word
        ↓
计算 2t 个 syndrome
        ↓
所有 syndrome 为 0？
   ├─ 是：NO_ERROR
   └─ 否：
        Berlekamp-Massey
              ↓
        错误位置多项式 Λ(x)
              ↓
        Chien search
              ↓
        root count 检查
              ↓
        翻转错误位置
              ↓
        post-syndrome 检查
              ↓
        提取母码系统位
              ↓
        删除虚拟缩短位
              ↓
        恢复 payload
```

## 6.2 syndrome 序列

对于接收多项式：

$$
r(x)
$$

计算：

$$
S_j=r(\alpha^j),
\qquad j=1,2,\ldots,2t
$$

输出必须保留：

```text
syndrome sequence
allZeroSyndrome
nonzeroSyndromeCount
```

便于 MATLAB 中间量对比。

## 6.3 Berlekamp-Massey 输出

BM 至少输出：

```text
errorLocatorPolynomial
locatorDegree
discrepancy history
iteration count
```

错误位置多项式：

$$
\Lambda(x)
=
1+\Lambda_1x+\cdots+\Lambda_\nu x^\nu
$$

其中：

$$
\nu\le t
$$

## 6.4 Chien 搜索

对每个可能位置测试：

$$
\Lambda(\alpha^{-i})=0
$$

必须统一：

- 位置 0 对应哪个多项式幂；
- 系统位和校验位的方向；
- MATLAB root index 与 C++ bit index 的转换；
- 缩短虚拟位置是否允许出现错误 root。

## 6.5 译码状态

建议：

```text
NO_ERROR
CORRECTED
DECODE_FAILURE
LOCATOR_DEGREE_EXCEEDS_T
INVALID_ROOT_COUNT
ROOT_OUT_OF_RANGE
ROOT_IN_SHORTENED_POSITION
POST_SYNDROME_NONZERO
INVALID_CONFIGURATION
```

## 6.6 纠错能力测试

对于每个 Case：

```text
0 error
1 error
2 errors
...
t errors
```

必须覆盖：

- payload system 位；
- parity 位；
- 首尾边界；
- 混合位置；
- 随机位置；
- 最大纠错能力位置集合。

正式全组合可能规模过大，因此采用：

```text
0/1 error 全位置覆盖
2..t error 分层固定 seed
边界组合
随机组合
```

并让 MATLAB 使用完全相同的 error seed。

## 6.7 超出能力测试

对：

```text
t+1
t+2
高权重随机错误
```

进行审计，但不能要求全部检测失败。

分类：

```text
译码失败
post-check 失败
误纠为另一合法码字
payload 恰好恢复
root 数异常
root 落入缩短虚拟位
```

# 十、BCH-17：突发错误与交织

交织必须保持为独立测试项，不能混入普通 AWGN 基线。

建议实验顺序：

```text
突发错误模型冻结
→ 交织器互逆验证
→ 无交织基线
→ 块交织
→ 行列交织
→ 固定 seed 伪随机交织
→ FER 改善和附加时延
```

重点研究分块 BCH：

```text
无交织：
连续错误集中在少数 BCH(15,11,1) 子块，
单块错误数容易超过 t=1。

交织后：
连续错误被分散到不同子块，
可能使每个子块只出现 0 或 1 个错误。
```

实验维度建议：

```text
Payload：200、300
编码：分块、整块
交织：无、块、行列、伪随机
突发长度：1、2、4、8、16、32
代表信道强度：2～3 个
```
