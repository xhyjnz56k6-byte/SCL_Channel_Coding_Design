# 信道编码仿真项目参数与实施计划

## 1. 项目目标

面向 200 bit 和 300 bit 随机信息帧，分别完成 BCH、卷积码和基于 5G NR BG2 的 Direct QC-LDPC 编译码仿真，并在统一信道条件下比较：

- BER、FER、译码成功率；
- 编码后码长、有效码率；
- 编码增益；
- 平均/最大译码时延；
- 算法复杂度；
- 交织对突发错误的改善效果。

核心编译码算法使用基础 C++ 实现；Python负责驱动编译、测试、数据生成、结果检查和绘图；MATLAB用于独立参考验证。

---

## 2. 统一仿真条件

| 项目 | 计划配置 |
|---|---|
| 输入信息 | 随机生成 200 bit 或 300 bit |
| 调制 | BPSK，`0 -> +1`，`1 -> -1` |
| 基础信道 | AWGN |
| 扩展信道 | 多径、频偏、多普勒、短时遮挡、突发错误 |
| 软信息 | `LLR = 2y / sigma^2` |
| 随机性 | 公共信息帧池、公共标准高斯母噪声、固定 seed |
| 正式指标 | BER、FER、成功率、码长、有效码率、编码增益、平均/最大译码时延、复杂度 |
| 仿真流程 | noiseless → MATLAB 对比 → smoke → prescan → formal |
| 停止规则 | `(达到最小帧数且达到目标误帧数) 或 达到最大帧数` |

有效码率：

\[
R_{\mathrm{eff}}=\frac{K_{\mathrm{payload}}}{N_{\mathrm{tx}}}
\]

AWGN 噪声方差：

\[
\sigma^2=\frac{1}{2R_{\mathrm{eff}}10^{E_b/N_0/10}}
\]

编码增益统一在指定目标误码点计算，例如：

\[
G_{\mathrm{coding}}
=
(E_b/N_0)_{\mathrm{reference}}
-
(E_b/N_0)_{\mathrm{candidate}}
\]

优先在 `FER = 10^-2` 或双方曲线均覆盖的统一目标点进行比较。

## 3. BCH 参数计划

### 3.1 分块 BCH

| 参数 | 200 bit | 300 bit |
|---|---:|---:|
| 码型 | BCH(15,11,1) | BCH(15,11,1) |
| 每组信息位 | 11 bit | 11 bit |
| 分组数 | 19 | 28 |
| 填充位 | 9 bit | 8 bit |
| 编码后总长度 | 285 bit | 420 bit |
| 有效码率 | 0.702 | 0.714 |
| 生成多项式 | `g(x)=x^4+x+1` | 同左 |
| 编码结构 | 系统码 | 系统码 |
| 译码算法 | syndrome 查表译码 | 同左 |

### 3.2 整块 BCH

| 输入 | 计划码型 | 输出码长 | 译码算法 |
|---|---|---:|---|
| 200 bit | 缩短 BCH(255,207) | 248 bit | BM + Chien 搜索 |
| 300 bit | 缩短 BCH(511,421) | 390 bit | BM + Chien 搜索 |
| 扩展对比 | 缩短 BCH(511,385) | 按缩短规则确定 | BM + Chien 搜索 |

### 3.3 主要测试

1. BCH(15,11,1) 全部 2048 个输入遍历；
2. 15 个位置逐位单错注入；
3. 200/300 bit 分块、填充和恢复；
4. 与 MATLAB BCH 官方函数逐帧对比；
5. AWGN 下 BER、FER、编码增益和译码时延；
6. 分块 BCH 与整块 BCH 对比；
7. 后续开展交织和突发错误测试。

## 4. 卷积码参数计划

### 4.1 基本参数

| 参数 | 配置 |
|---|---|
| 输入长度 | 300 bit 为主，程序兼容 200 bit |
| 母码率 | 1/2 |
| 约束长度 | K = 7 |
| 生成多项式 | G1 = 171(oct)，G2 = 133(oct) |
| 状态数 | 64 |
| 初始状态 | 全零 |
| 整块终止 | 加 6 个尾比特回零 |
| 译码算法 | 硬判决 Viterbi、软判决 Viterbi |
| 扩展码率 | 2/3、3/4 打孔 |
| 分块方案 | 连续编码 + 滑窗 Viterbi |

### 4.2 整块输出长度

| 输入 | 1/2 码率输出 | 有效码率 |
|---|---:|---:|
| 200 bit | 412 bit | 0.485 |
| 300 bit | 612 bit | 0.490 |

300 bit 条件下，2/3 和 3/4 打孔输出约 459 bit 和 408 bit，最终长度由冻结后的打孔图样确定。

### 4.3 主要测试

1. trellis 与 MATLAB `poly2trellis` 对比；
2. 编码输出与 MATLAB `convenc` 对比；
3. 硬判决、软判决 Viterbi 与 MATLAB `vitdec` 对比；
4. 1/2、2/3、3/4 码率 BER、FER、编码增益和时延对比；
5. 回溯深度、软信息量化和滑窗长度对比；
6. 整块与连续分块方案对比；
7. 后续开展交织和复杂信道测试。

## 5. Direct QC-LDPC 参数计划

### 5.1 固定原则

| 参数 | 配置 |
|---|---|
| 基图 | 只采用 BG2 |
| 输入长度 | 200 bit 或 300 bit |
| 编码结构 | 基于 BG2 切割的 Direct QC-LDPC |
| 提升因子 | 从合法 Zc 中动态选择 |
| 目标码长 | **480 bit、576 bit、以及不超过 640 bit 的扩展码长** |
| 码长选择 | 优先选取可由合法 Zc 和切割列数实现、且最接近目标值的码长 |
| 填充 | 使系统位容量不小于输入长度 |
| 总码长上限 | 640 bit（LDPC 本轮计划） |
| 编码算法 | Direct 编码，GF(2) 求解校验位 |
| 调度方式 | Layered |
| 译码算法 | **Layered BP 与 Layered NMS 对比** |
| BP | SPA/tanh-atanh 更新 |
| NMS | Min-Sum + 归一化系数 alpha |
| 最大迭代次数 | 10、20、30，必要时保留 32 作为工程对照 |
| 提前停止 | 每轮结束后检查 syndrome |
| 交织 | 不配置 |
|  |                                                          |
| MATLAB 对比 | 对同一 BG2 子矩阵建立独立参考实现 |

### 5.2 目标码长与 case 设计

针对 300 bit 主场景，构造：

| Case | 目标码长 | 目标码率 | 用途 |
|---|---:|---:|---|
| LDPC-480 | 480 bit | 300/480 = 0.625 | 对比卷积码 2/3 附近方案 |
| LDPC-576 | 576 bit | 300/576 ≈ 0.521 | 对比卷积码 1/2 方案 |
| LDPC-EXT | ≤640 bit | 可配置 | 短码扩展对比 |

若合法 Zc 与 BG2 切割结构无法精确得到目标码长，则选择最接近目标的可实现码长，并在结果中明确记录：

```text
目标码长、实际码长、Zc、kb、nb、mb、Keff、filler、
N、M、实际码率、有效码率、Hp rank
```

### 5.3 BP 与 NMS 对比

在相同 BG2 子矩阵、相同输入、相同噪声和相同最大迭代次数下比较：

| 对比项 | BP | NMS |
|---|---|---|
| 校验节点更新 | tanh/atanh SPA | Min-Sum × alpha |
| 调度 | Layered | Layered |
| 提前停止 | syndrome | syndrome |
| 主要参数 | 最大迭代次数、LLR 裁剪 | alpha、最大迭代次数、LLR 裁剪 |
| 主要输出 | BER、FER、编码增益、迭代次数、时延 | 同左，并统计相对 BP 的性能损失与复杂度下降 |

NMS 归一化系数先通过 prescan 预扫描，例如：

```text
alpha ∈ {0.70, 0.75, 0.80, 0.85, 0.90}
```

再冻结正式参数。

### 5.4 主要测试

1. BG2 表、Zc 和循环移位检查；
2. Direct 子矩阵尺寸与 `Hp` 满秩检查；
3. 编码后验证 `H*c^T = 0`；
4. BP、NMS 无噪声译码；
5. C++ 与 MATLAB 的矩阵、码字、译码结果、syndrome 和迭代次数对比；
6. 480、576、≤640 bit 三类码长的 BER、FER 和编码增益对比；
7. BP 与 NMS 的性能、迭代次数、平均/最大时延和复杂度对比；
8. 复杂信道测试，LDPC 不配置交织。

## 6. 公共帧池与母噪声

### 6.1 公共帧池

- 分别建立 200 bit 和 300 bit 帧池；
- 三类编码读取相同输入帧；
- 帧池按 shard 追加，不修改旧帧；
- 最大支持 50000 帧；
- smoke 读取前 100 帧；
- prescan 最多读取前 2000 帧；
- formal 按停止规则最多读取 50000 帧；
- 记录 master seed、frame index 和文件哈希。

### 6.2 公共标准高斯母噪声

每帧生成：

\[
z_i\sim\mathcal N(0,1)
\]

各编码按自己的码长截取前 N 个样本，并按各自有效码率计算噪声强度：

\[
y=x+\sigma z
\]

验证阶段保存固定噪声；formal 阶段根据 seed、SNR 和 frame index 确定性生成。

## 7. 仿真停止规则

| 阶段 | 最小帧数 | 目标误帧数 | 最大帧数 | 用途 |
|---|---:|---:|---:|---|
| smoke | 20 | 5 | 100 | 检查链路 |
| prescan | 300 | 30 | 2000 | 定位 waterfall |
| formal trial | 5000 | 100 | 20000 | 冻结正式参数 |
| formal | 5000 | 200 | 50000 | 正式结果 |

正式 SNR 步长计划为 0.1～0.2 dB，最终根据 prescan 结果冻结。

---

## 8. 正式输出指标

所有正式实验至少输出：

- BER；
- FER；
- 译码成功率；
- 编码后码长；
- 有效码率；
- **编码增益**；
- 平均译码时延；
- 最大译码时延；
- 算法复杂度；
- 停止原因和实际帧数。

此外：

- BCH：误纠率、分块/整块对比；
- 卷积码：硬/软判决、回溯深度、滑窗时延；
- LDPC：BP/NMS、平均迭代次数、最大迭代次数命中率、提前停止比例、NMS 相对 BP 的性能损失和复杂度下降；
- 交织实验：FER 改善量和附加时延。

编码增益优先在统一目标 FER 点计算；若曲线未覆盖目标点，则注明“无法计算”，不得强行外推。

## 9. 实施顺序

### 第一阶段：公共基础

1. 冻结参数与目录；
2. 建立 200/300 bit 公共帧池；
3. 建立公共标准高斯母噪声；
4. 统一 BPSK、AWGN、LLR、BER、FER 和编码增益定义；
5. 建立 Python 驱动和结果检查脚本。

### 第二阶段：BCH

1. BCH(15,11,1) 编码与 syndrome 译码；
2. 200/300 bit 分块、填充和恢复；
3. MATLAB 对比；
4. noiseless、smoke、prescan、formal；
5. 整块缩短 BCH；
6. 分块/整块和交织对比。

### 第三阶段：卷积码

1. trellis 和 1/2 零尾编码；
2. 硬判决、软判决 Viterbi；
3. MATLAB 对比；
4. 2/3、3/4 打孔；
5. smoke、prescan、formal；
6. 连续编码与滑窗 Viterbi；
7. 交织和复杂信道。

### 第四阶段：LDPC

1. BG2、合法 Zc 和目标码长映射；
2. 构造 480、576、≤640 bit 候选 Direct 子矩阵；
3. `Hp` 秩检查与 Direct 编码；
4. Layered BP；
5. Layered NMS 与 alpha 预扫描；
6. MATLAB 独立参考验证；
7. noiseless、smoke、prescan、formal；
8. BP/NMS、三种码长和复杂信道对比。

### 第五阶段：综合比较

输出：

- BER/FER 曲线；
- 编码增益曲线或表格；
- 码长、码率和时延表；
- BP/NMS 与硬/软 Viterbi 对比；
- 交织改善结果；
- 三类编码适用场景和最终建议。
