# 信道编码仿真项目参数与实施计划

## 1. 项目目标

项目面向 200 bit 和 300 bit 随机信息帧，分别完成 BCH 码、卷积码和基于 5G NR BG2 的 Direct QC-LDPC 编译码仿真，并在统一信道条件下比较：

- BER、FER；
- 编码后码长与有效码率；
- 译码成功率；
- 平均/最大译码时延；
- 算法复杂度；
- 交织对突发错误的改善效果。

核心编译码算法采用基础 C++ 实现；Python负责驱动编译、测试、数据生成和绘图；MATLAB用于独立参考验证。

---

## 2. 统一仿真条件

| 项目 | 计划配置 |
|---|---|
| 输入信息 | 随机生成 200 bit 或 300 bit |
| 调制 | BPSK，`0 -> +1`，`1 -> -1` |
| 基础信道 | AWGN |
| 扩展信道 | 多径、频偏、多普勒、短时遮挡、突发错误 |
| 软信息 | `LLR = 2y / sigma^2` |
| 随机性 | 公共信息帧池、公共标准高斯母噪声、固定 seed |
| 基础指标 | BER、FER、成功率、码长、有效码率、译码时延 |
| 仿真阶段 | noiseless → MATLAB 对比 → smoke → prescan → formal |
| formal 停止规则 | `(帧数达到最小值且误帧数达到目标值) 或 达到最大帧数` |

有效码率统一按：

\[
R_{\mathrm{eff}}=\frac{K_{\mathrm{payload}}}{N_{\mathrm{tx}}}
\]

计算。

AWGN 噪声方差统一按：

\[
\sigma^2=\frac{1}{2R_{\mathrm{eff}}10^{E_b/N_0/10}}
\]

计算。

## 3. BCH 码参数计划

### 3.1 研究对象

BCH 码用于 200 bit 和 300 bit 低速信息，比较分块方案与整块缩短 BCH 方案。

### 3.2 分块 BCH

| 参数 | 200 bit | 300 bit |
|---|---:|---:|
| 码型 | BCH(15,11,1) | BCH(15,11,1) |
| 每组信息位 | 11 bit | 11 bit |
| 分组数 | 19 | 28 |
| 填充位 | 9 bit | 8 bit |
| 编码后总长度 | 285 bit | 420 bit |
| 有效码率 | 200/285 ≈ 0.702 | 300/420 ≈ 0.714 |
| 生成多项式 | `g(x)=x^4+x+1` | 同左 |
| 编码结构 | 系统码 | 系统码 |
| 译码算法 | syndrome 查表，单比特纠错 | 同左 |

### 3.3 整块 BCH

| 输入 | 计划码型 | 输出码长 | 译码算法 |
|---|---|---:|---|
| 200 bit | 缩短 BCH(255,207) | 248 bit | BM + Chien 搜索 |
| 300 bit | 缩短 BCH(511,421) | 390 bit | BM + Chien 搜索 |
| 扩展对比 | 缩短 BCH(511,385) | 按缩短规则确定 | BM + Chien 搜索 |

### 3.4 BCH 主要测试

1. BCH(15,11,1) 全部 2048 个输入遍历测试；
2. 15 个位置逐位单错注入；
3. 无噪声 200/300 bit 分块恢复；
4. 与 MATLAB BCH 官方函数逐帧对比；
5. AWGN 下 BER、FER 和时延测试；
6. 后续加入块交织、行列交织和伪随机交织；
7. 比较分块 BCH 与整块缩短 BCH。

## 4. 卷积码参数计划

### 4.1 基本参数

| 参数 | 配置 |
|---|---|
| 输入长度 | 300 bit 为主，程序兼容 200 bit |
| 母码率 | 1/2 |
| 约束长度 | K = 7 |
| 生成多项式 | G1 = 171(oct)，G2 = 133(oct) |
| 状态数 | 64 |
| 初始状态 | 全零状态 |
| 整块终止 | 加入 6 个尾比特回零 |
| 译码算法 | 硬判决 Viterbi、软判决 Viterbi |
| 扩展码率 | 2/3、3/4 打孔 |
| 分块方案 | 连续编码 + 滑窗 Viterbi |

### 4.2 整块输出长度

| 输入 | 1/2 码率输出 | 有效码率 |
|---|---:|---:|
| 200 bit | 2×(200+6)=412 bit | 200/412 ≈ 0.485 |
| 300 bit | 2×(300+6)=612 bit | 300/612 ≈ 0.490 |

300 bit 条件下，2/3 和 3/4 打孔方案计划输出约 459 bit 和 408 bit，具体长度由冻结后的打孔图样确定。

### 4.3 卷积码主要测试

1. 64 状态 trellis 与 MATLAB `poly2trellis` 对比；
2. 编码输出与 MATLAB `convenc` 对比；
3. 无噪声硬判决、软判决 Viterbi 恢复；
4. 与 MATLAB `vitdec` 逐帧对比；
5. 1/2、2/3、3/4 码率 AWGN 性能对比；
6. 回溯深度、滑窗长度与时延对比；
7. 整块与连续分块方案对比；
8. 后续进行交织和突发错误测试。

## 5. Direct QC-LDPC 参数计划

### 5.1 固定原则

| 参数 | 配置 |
|---|---|
| 基图 | BG2 |
| 输入长度 | 200 bit 或 300 bit |
| 编码结构 | 基于 BG2 切割的 Direct QC-LDPC |
| 提升因子 | 从合法 Zc 中动态选择 |
| 填充 | 使系统位容量不小于输入长度 |
| 输出码长 | 根据 Zc、保留列数和目标码率动态确定 |
| 总码长约束 | 不超过 1000 bit |
| 编码算法 | Direct 编码，GF(2) 求解校验位 |
| 译码调度 | Layered |
| 译码算法 | Layered SPA/BP |
| 最大迭代次数 | 默认 32，可配置 10、20、30、32 |
| 提前停止 | 每轮完成后检查 syndrome |
| 交织 | 不配置 |
| 标准速率匹配 | 不采用 |
| MATLAB 对比 | 对同一 BG2 子矩阵建立独立参考实现 |

### 5.2 参数选择机制

1. 固定 BG2；
2. 输入 `K = 200` 或 `K = 300`；
3. 枚举合法提升因子 `Zc`；
4. 选择能够容纳输入信息的最小合适 `Zc`；
5. 根据目标码率选择保留的基图列；
6. 构造 Direct 子矩阵 `H=[Hu Hp]`；
7. 检查 `Hp` 满秩；
8. 计算实际码长、填充位和有效码率；
9. 编码后检查 `H*c^T=0`；
10. 进入 Layered BP 译码。

每个 LDPC case 必须记录：

```text
BG2、Zc、kb、nb、mb、K、Keff、filler、N、M、
实际码率、有效码率、Hp rank、最大迭代次数
```

### 5.3 LDPC 主要测试

1. BG2 表和循环移位检查；
2. Zc 自动选择测试；
3. Direct 子矩阵维度与 `Hp` 秩检查；
4. 编码码字 syndrome 检查；
5. 无噪声译码；
6. 固定扰动和固定噪声逐帧验证；
7. C++ 与 MATLAB 的矩阵、码字、译码结果、syndrome 和迭代次数对比；
8. AWGN 下 BER、FER、平均迭代次数和平均/最大译码时延测试。

## 6. 公共帧池与噪声机制

### 6.1 公共帧池

- 分别建立 200 bit 和 300 bit 公共信息帧池；
- BCH、卷积码和 LDPC 读取相同输入帧；
- 帧池按 shard 追加，只增加新帧，不修改旧帧；
- 计划最大支持 50000 帧；
- smoke 可读取前 100 帧，prescan 读取前 2000 帧，formal 按停止规则最多读取 50000 帧；
- 记录 master seed、frame index 和文件哈希，保证复现。

### 6.2 公共标准高斯母噪声

每帧生成：

\[
z_i\sim\mathcal N(0,1)
\]

不同编码按自己的码长截取前 N 个样本，再根据各自有效码率计算 `sigma`：

\[
y=x+\sigma z
\]

验证阶段保存固定噪声文件；formal 阶段根据 seed、SNR 和 frame index 确定性生成。

## 7. 仿真停止规则

### 7.1 smoke

| 参数 | 建议值 |
|---|---:|
| 最小帧数 | 20 |
| 目标误帧数 | 5 |
| 最大帧数 | 100 |

用途：验证程序能否正确运行，不用于正式结论。

### 7.2 prescan

| 参数 | 建议值 |
|---|---:|
| 最小帧数 | 300 |
| 目标误帧数 | 30 |
| 最大帧数 | 2000 |
| SNR 步长 | 约 0.5 dB |

用途：定位 waterfall 区域和估算运行时间。

### 7.3 formal trial

| 参数 | 建议值 |
|---|---:|
| 最小帧数 | 5000 |
| 目标误帧数 | 100 |
| 最大帧数 | 20000 |

用途：检查正式仿真帧数是否合理。

### 7.4 formal

初步计划：

| 参数 | 建议值 |
|---|---:|
| 最小帧数 | 5000 |
| 目标误帧数 | 200 |
| 最大帧数 | 50000 |
| SNR 步长 | 0.1～0.2 dB |

最终数值根据 smoke、prescan 和 formal trial 的误帧密度及运行时间冻结。

## 8. 实施顺序

### 第一阶段：公共基础

1. 冻结目录结构和参数格式；
2. 建立 200/300 bit 公共帧池；
3. 建立公共标准高斯母噪声机制；
4. 统一 BPSK、AWGN、LLR、BER、FER 和停止规则；
5. 建立 Python 驱动和结果检查脚本。

### 第二阶段：BCH

1. BCH(15,11,1) 单块编码；
2. syndrome 查表译码；
3. 200/300 bit 分块与填充；
4. MATLAB 对比；
5. noiseless、smoke、prescan、formal；
6. 整块缩短 BCH；
7. BCH 分块与整块比较；
8. 交织和复杂信道。

### 第三阶段：卷积码

1. trellis；
2. 1/2 零尾编码；
3. 硬判决 Viterbi；
4. 软判决 Viterbi；
5. MATLAB 对比；
6. 2/3、3/4 打孔；
7. smoke、prescan、formal；
8. 连续编码与滑窗 Viterbi；
9. 交织和复杂信道。

### 第四阶段：LDPC

1. BG2 表和合法 Zc；
2. 动态 Zc 选择；
3. BG2 子矩阵切割；
4. `Hp` 秩检查与 Direct 编码；
5. Layered SPA/BP；
6. MATLAB 独立参考验证；
7. noiseless、smoke、prescan、formal；
8. 复杂信道测试，不配置交织。

### 第五阶段：综合比较

输出：

- BER/FER 曲线；
- 码长和有效码率表；
- 平均/最大译码时延；
- 卷积码回溯与 LDPC 迭代统计；
- 交织改善结果；
- 三类编码适用场景和最终建议。
